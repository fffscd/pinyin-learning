# 幼儿园中班拼音学习小游戏需求规划说明书

## 1. 项目目标

面向幼儿园中班儿童，设计一个轻量、可操作、反馈友好的拼音启蒙小游戏。产品重点放在听辨、认读、口型模仿和兴趣建立，避免过早引入高强度书写、考试化拼读和复杂规则讲解。

## 2. 目标用户

- 主要用户：幼儿园中班儿童，年龄约 4 到 5 岁。
- 辅助用户：家长、老师，用于陪伴学习、课堂互动或课后练习。
- 使用场景：平板、电脑浏览器、互动白板；每次学习建议 5 到 10 分钟。

## 3. 学习范围

### 3.1 首期内容

- 单韵母：`a`、`o`、`e`、`i`、`u`、`ü`。
- 声母入门：`b`、`p`、`m`、`f`、`d`、`t`、`n`、`l`。
- 四声感知：通过听音和动画感受声调变化，不要求孩子独立书写声调。
- 简单联想词：例如 `a` 对应“阿姨”、`m` 对应“妈妈”等，内容需贴近日常生活。

### 3.2 暂不纳入首期

- 复杂拼读规则。
- 全量声母、韵母、整体认读音节。
- 拼音书写笔顺训练。
- 排名、竞赛、惩罚式失败机制。

## 4. 核心玩法

### 4.1 听音找拼音

系统播放拼音读音，孩子从 2 到 3 个大按钮中选择正确拼音。

- 每题只有一个正确答案。
- 初始阶段使用 2 个选项，熟悉后增加到 3 个选项。
- 答对后播放正向反馈动画和鼓励语音。
- 答错后给出温和提示，再播放一次读音。

### 4.2 拼音配图

展示一个生活化图片或插画，孩子选择对应的拼音卡片。

- 图片必须清晰、单一主体、便于幼儿理解。
- 每个拼音搭配 1 到 2 个固定示例词。
- 不追求词语拼读准确输出，重点是建立声音和符号的关联。

### 4.3 声调小滑梯

用四条不同走势的轨迹表现一声、二声、三声、四声，孩子点击轨迹听对应声调。

- 以 `a`、`o`、`e` 作为首批声调感知对象。
- 通过动画表现音高变化。
- 不要求孩子判断复杂音节声调。

### 4.4 拼音小火车

孩子完成若干题后获得一节车厢，车厢上显示已学习拼音。

- 每 3 到 5 题形成一次阶段反馈。
- 学习完成后展示“今日小火车”。
- 只记录学习进度，不做公开排名。

## 5. 页面与流程

### 5.1 首页

- 展示游戏标题和 3 个入口：开始学习、自由练习、学习记录。
- 默认突出“开始学习”。
- 控件尺寸要适合幼儿点击。

### 5.2 关卡选择

- 按学习主题分组：单韵母、声母入门、声调感知。
- 每组显示完成进度。
- 未解锁内容可以弱化显示，但不制造挫败感。

### 5.3 游戏页

- 上方显示当前进度，例如第 2/5 题。
- 中间显示题目主体：音频按钮、图片或声调轨迹。
- 下方显示可点击选项。
- 页面保留“再听一次”和“返回”按钮。

### 5.4 结算页

- 展示本轮学习到的拼音。
- 展示完成星星或车厢等非竞争奖励。
- 提供“再玩一次”和“换一组”。

## 6. 交互与体验要求

- 文字大、对比清晰、按钮间距充足。
- 每屏信息量少，减少干扰元素。
- 反馈要即时，动画时长控制在 1 秒左右。
- 答错提示使用“再听一听”“找找这个声音”这类温和表达。
- 支持重复听音。
- 支持静音模式，但默认开启声音。
- 避免长段说明文字，优先使用图标、语音和动画。

## 7. 内容与素材要求

- 拼音读音需要清晰、标准、音量一致。
- 图片或插画需符合幼儿审美，主体明确，不使用恐吓、暴力或过度刺激内容。
- 示例词优先选取家庭、动物、食物、生活用品、自然场景。
- 如暂时没有真人音频，首版可使用浏览器语音合成作为占位，并预留替换真人音频的结构。

## 8. 数据与进度

首期只使用本机临时 JSON 文件保存记录。

- 记录已完成关卡。
- 记录每个拼音的练习次数和最近一次结果。
- 记录静音开关。
- 不采集姓名、手机号、定位、设备标识等个人信息。

## 9. 技术规划

在项目为空目录的前提下，建议首期做成单页 Web 小游戏。

- 前端框架：React 或纯 HTML/CSS/JavaScript 均可。
- 推荐方案：Vite + React + TypeScript，便于后续扩展关卡和组件。
- 样式：CSS Modules 或普通 CSS，优先保证简单可维护。
- 音频：当前实现使用本地音频文件，真人录音按 `assets/audio/README.md` 的命名规则放置。
- 存储：本地 Node 服务写入临时 JSON 文件。
- 部署：可作为静态站点运行。

## 10. 首期 MVP 范围

首期建议只实现一条完整学习闭环：

1. 首页。
2. 单韵母关卡。
3. 听音找拼音玩法。
4. 5 题一轮。
5. 答题即时反馈。
6. 结算页。
7. 本地记录完成状态。

## 11. 后续迭代方向

- 补齐真人录制音频素材。
- 增加老师或家长可查看的学习记录。
- 增加课堂大屏模式。
- 增加离线可用能力。

## 12. 验收标准

### 12.1 功能验收

- 用户可以从首页进入一轮学习。
- 用户可以听到拼音读音。
- 用户可以点击选项答题。
- 答对和答错都有明确反馈。
- 一轮完成后进入结算页。
- 刷新页面后仍能看到已完成状态。

### 12.2 体验验收

- 中班儿童在成人陪伴下可以理解主要操作。
- 主要按钮在平板和桌面端都易于点击。
- 单轮学习时长不超过 10 分钟。
- 答错不会出现负面、惩罚、嘲讽类提示。

### 12.3 安全与隐私验收

- 不包含密钥、令牌或个人身份信息。
- 不上传用户数据。
- 不依赖登录。
- 不提交系统生成文件或非必要大型二进制文件。

## 13. 待确认问题

1. 首期是否只做单韵母 `a/o/e/i/u/ü`？
   - 推荐：是。先降低学习负担，保证一条完整闭环做扎实。
2. 是否需要中文语音提示，例如“请找到 a”？
   - 推荐：需要。中班儿童更容易理解任务目标。
3. 视觉风格偏向哪一种：课堂清爽、卡通童趣、绘本柔和？
   - 推荐：卡通童趣。按钮大、颜色明快，但控制动画和装饰数量。
4. 是否需要适配手机，还是优先平板和电脑？
   - 推荐：优先平板和电脑，同时保证手机基础可用。
5. 首版是否接受浏览器语音合成，后续再替换真人音频？
   - 推荐：接受。首版先验证玩法和流程，音频结构预留替换能力。

## 14. 审核确认项

如按推荐方案进入开发，首期实现范围为：

1. 单页 Web 小游戏。
2. 单韵母 `a/o/e/i/u/ü`。
3. 听音找拼音玩法。
4. 中文语音提示和重复听音。
5. 5 题一轮，完成后展示结算页。
6. 本地保存完成状态和静音设置。
7. 优先适配平板和电脑，兼顾手机基础布局。

审核回复建议：

- 回复“通过”：按以上范围开始开发。
- 回复“修改：...”：先调整说明书，再等待确认。

## 15. 建议实施顺序

1. 确认本说明书范围。
2. 初始化 Web 项目。
3. 建立拼音题库数据结构。
4. 实现首页、游戏页、结算页。
5. 接入读音播放和答题反馈。
6. 增加本地进度记录。
7. 做桌面和平板视口检查。
