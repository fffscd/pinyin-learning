# 家长周报 设计

日期：2026-06-11
状态：方向已确认，细节待评审；建议先实施 p15（家长区）

## 背景

记录页现有累计统计、每日历史（`recentDailyStats`）与「已掌握 / 复习中 / 易忘」分布（`masteryBreakdown`），是面向数据的视图。家长真正需要的是「这周孩子学了什么、卡在哪、我能做什么」的行动指引。家长的线下配合是巩固效果最大的零成本杠杆。

## 目标

家长区顶部增加「本周摘要」区块：本周所学、易忘清单、生活配合建议，全部由现有 progress 数据纯前端生成。

## 设计

### 数据汇总（新增 `buildWeeklyReport()`）

取最近 7 个自然日（`getLocalDateId` 回溯）：

1. 本周新学：7 日内课表引入的 `newItems` 并集（由 dayIndex 区间映射 `getDailyPlan` 取得）。
2. 练习概况：7 日 `dailyStats` 汇总——完成课程天数、总题数、正确率（`formatPercent` 复用）。
3. 易忘清单：`masteryBreakdown` 的易忘集合 ∩ 已解锁项，按 box 升序最多列 5 个。
4. 抽查记录（若 p05 已实施）：本周抽查次数与结果。

### 生活配合建议（模板生成）

- 每个 `PINYIN_ITEMS` 条目已有 `word`（生活例词），据此套模板生成建议句，如：
  - 韵母/声母：「这周学了 {label}，生活里见到{word}时可以指给孩子：{word} 的 {label}。」
  - 易忘项：「{label} 最近容易忘，散步时玩『找声音』：谁先发现一个 {label} 开头的东西。」
- 模板常量 `WEEKLY_TIP_TEMPLATES` 按条目类型（initial/final/whole/syllable）分组，每周从匹配模板中确定性抽取（`seededRandom` 以周一日期为种子，同一周内容稳定）。
- 建议最多 3 条，避免给家长布置负担。

### 呈现

- 位置：`recordsView` 门禁后的首屏第一个区块，`parent-theme` 成人样式（p15）。
- 结构：一段概况文字 + 易忘项小表 + 建议列表。无图表库，正确率用纯 CSS 横条即可。
- 数据不足 7 天（新用户）：显示「开课第 N 天」与已有天数的摘要，不留空白。

## 不做的事（YAGNI）

- 不做导出/打印/推送（本地静态应用，无后端）。
- 不做周环比曲线（数据量小，意义有限）。
- 不做 LLM 生成个性化建议（违反无依赖约束）。

## 影响的文件

- `app.js`：`buildWeeklyReport`、`WEEKLY_TIP_TEMPLATES`、`recordsView` 区块。
- `styles.css`：周报区块、横条。
- `scripts/verify-goal.js`：周报汇总函数的单元断言（构造 7 日数据验证并集与易忘排序）。

## 验证方式

- headless：构造跨周数据断言新学并集、易忘 top5、同周建议稳定性；新用户第 2 天的降级显示。
- 人工：家长区周报可读性（请一位真实家长读一遍，确认建议可执行）。
