# 引导角色（火车头拟人化） 设计

日期：2026-06-11
状态：方向已确认，细节待评审；依赖角色提示音录制

## 背景

产品名为「拼音小火车」，`trainArt()` 已有火车视觉，但它只是静态装饰。提示音（`AUDIO_PROMPTS` 的 correct/retry/complete 等）是无来源的系统音，开场、收尾没有承接者。

推测依据：拟人化引导者能提升幼儿对指令的依从度与情感联结，是幼教产品的通行做法；同一角色发出鼓励与安抚，比无来源提示更能稳定情绪。通行结论，未逐条核验文献。

## 目标

把小火车头升级为贯穿全程的引导角色「嘟嘟」（名字实施时可定）：有表情状态，承担开场问候、答对庆祝、答错安抚、到站收尾四类时刻；全部提示音统一为同一音色。

## 设计

### 角色视觉

- 基于 `trainArt()` 改造：火车头加眼睛与嘴巴，SVG 内用 CSS 类切换表情状态：
  - `idle`：平静微笑（默认，出现在首页大卡、游戏页角落）。
  - `cheer`：眼睛弯弯 + 冒蒸汽（答对、结算落星时）。
  - `soothe`：温和眨眼（答错时，配合 p10 的重试流程）。
  - `arrive`：鸣笛冒彩烟（到站收尾，配合 p01、p14）。
- 游戏页中角色以小尺寸常驻角落（避开星星托盘与返回按钮），状态随答题切换；不抢答题区注意力。

### 角色语音

- 现有 `AUDIO_PROMPTS` 全套提示音用统一音色重制（MeloTTS 同一发音人参数，或后续真人录音，见 p20），新增：
  - `greet`：进入今日课程时的问候。
  - `arrive`：到站收尾。
- 语音内容短句化（2–4 秒），不依赖屏幕文字——壳层无文字约束不变，角色「说话」只走音频。
- 缺音频静默降级：表情动画照常，声音跳过。

### 接入点

- `startRound("lesson")`：播 `greet`，角色 `cheer` 一下再回 `idle`。
- `answer()` 答对/答错：表情切 `cheer`/`soothe`，与既有 correct/retry 音并轨（音频本身即角色声）。
- `nextQuestion()` 轮末与 p01 到站：`arrive` 表情 + 音频。
- `resultView`：角色在结算页主位，配合落星动画（衔接 p14）。

## 不做的事（YAGNI）

- 不做角色换装、命名自定义（换装由 p16 的点亮体系承担）。
- 不做对话气泡文字。
- 不做多角色。

## 影响的文件

- `app.js`：`trainArt`（表情参数化）、`startRound`、`answer`、`nextQuestion`、`gameView`、`resultView`、`AUDIO_PROMPTS`。
- `styles.css`：表情类切换、蒸汽/彩烟动画、reduced-motion 兜底。
- `assets/audio/`：greet、arrive 及全套提示音重制；`sync-audio-manifest.js` 收录。
- `scripts/generate-melotts-audio.py`：提示音批量生成参数。

## 验证方式

- headless：四种表情类在对应状态下正确挂载；缺 greet 音频时流程不阻塞。
- 人工：全流程走查表情切换时机；提示音音色一致性抽听；游戏页角色不遮挡答题区（窄屏重点）。
