# 儿童端星星激励 设计

日期：2026-06-03
状态：已与用户确认，待评审

## 背景

上一次迭代把"完成课程才解锁花园"的奖励门禁去掉后，孩子端失去了可见的进度/激励面（需求文档 4.4 节"拼音小火车车厢"本是非竞争性奖励）。需要补一个不重新引入门禁的儿童端激励。

用户想法：答对一题收集一颗小星星，星星累积放进盒子。

## 目标

让孩子每答对一题得到即时的星星反馈，星星在一轮内逐题收集，轮末倒进盒子，并在首页常驻展示终身累计总数，形成长期成就感。

## 约束

- **无文字界面**：项目刻意无文字、靠图标/图画，并有测试守护"导航壳层不出现可见中文长句"（6 个以上连续汉字判失败）。星星用图形表示，数量用阿拉伯数字（数字不触发该测试）。
- **无构建步骤、无第三方依赖**：纯 HTML/CSS/JS，沿用现有风格。
- **向后兼容**：旧本地存档没有星星字段，需默认补 0。
- **减少动画偏好**：尊重 `prefers-reduced-motion`，沿用项目已有写法。

## 确认的设计决策

经与用户逐项确认：

1. **盒子位置**：首页常驻盒子 + 结算页落星动画。
2. **累计方式**：永久累计，只增不减，不清零。
3. **过程收集**：游戏页角落星星托盘，每答对一题飞一颗星进托盘，逐题变多。
4. **数量表示**：用阿拉伯数字显示累计数（中班孩子可接受）。
5. **首页盒子**：可点击，打开放大浮层看星星堆 + 总数。

## 设计

### 数据（持久化）

- 新增 `state.progress.stars`：终身累计星星总数，整数，只增不减。
  - 在 `createEmptyProgress()` 中初始化为 0。
  - 在 `normalizeProgress()` 中：`stars: Number(saved.stars) || 0`，保证旧存档兼容。
  - 随 `saveProgress()` 一并写入本地存储/临时 JSON。
- 新增 `state.roundStars`：本轮临时计数，非持久化。
  - 在 `startRound()` 中重置为 0。
  - 用于游戏页托盘显示与结算页落星数量。

### 加星接入点（单点，全玩法统一）

所有玩法的答对判定都最终走 `answer()`：选项类题目经点击分发 `answer(answerId)`；花园拼花经 `selectBuildPart()` 调 `answer()`。因此只需在 `answer()` 的 `isCorrect` 分支接入：

- `state.roundStars += 1`
- `state.progress.stars += 1`
- 既有 `saveProgress()` 已在该函数内调用，无需新增。
- 触发"飞星进托盘"动画（见下）。

### 三处表现

1. **游戏页 · 角落星星托盘**
   - 游戏页角落（实现时定位，避开既有 `parent-corner`、`repeat-prompt`、返回首页按钮）放一个托盘元素：星星图标 + 本轮 `roundStars` 数字。
   - 答对时：舞台冒出一颗星，动画飞向托盘，托盘数字 +1。
   - 重渲染由现有 `render()` 负责更新数字；飞星为一次性瞬时元素或 CSS 动画类。

2. **结算页 · 落星进盒子**
   - 结算页展示盒子接收本轮 `roundStars` 颗星的落入动画，随后盒子显示新的累计 `stars` 总数。

3. **首页 · 常驻盒子**
   - 首页一角放星星盒图标 + 累计 `stars` 数字。
   - 点击 → 打开放大浮层：盒子里铺一堆星星（视觉上限封顶，避免上百颗）+ 累计数字；浮层可关闭。

### 视觉与无障碍

- 星星、盒子、托盘均用内联 SVG 或现有 `icon()` 体系新增图标。
- 数字用阿拉伯数字。盒子/托盘的 `aria-label` 可含中文（属性，不渲染为可见句子），与现有按钮用法一致。
- `prefers-reduced-motion: reduce` 时关闭飞星/落星动画，直接更新数字。

### 音效（YAGNI）

- 不为星星新增音效，沿用答对已有的 `correct` 音。
- 首页点盒子不加音效。

## 不做的事（YAGNI）

- 不做满盒换盒、每日清零、星星消费/兑换、排行榜。
- 不改题目生成、课程推进、各玩法内部逻辑。
- 不为星星新增音频资源。

## 影响的文件

- `app.js`：`createEmptyProgress`、`normalizeProgress`、`startRound`、`answer`、`gameView`（托盘）、`homeView`（盒子 + 点击放大）、`resultView`（落星）、点击分发（盒子放大浮层的开关）、可能新增 `icon()` 条目。
- `styles.css`：托盘、首页盒子、放大浮层、结算页落星与飞星动画、reduced-motion 兜底。
- `scripts/verify-child-shell.js`：新增断言。
- `README.md`：功能概览补充星星激励。

## 验证方式

- headless（`verify-child-shell.js`）：
  - 答对一题后 `state.progress.stars` 与 `state.roundStars` 各 +1。
  - `startRound` 后 `roundStars` 归零。
  - 游戏页渲染出托盘元素；首页渲染出盒子元素并显示累计数字；结算页显示累计数字。
  - 旧存档（无 stars 字段）经 `normalizeProgress` 后 `stars` 为 0。
  - 首页、花园、结算、游戏页均无可见中文长句（既有约束保持）。
- 浏览器截图：游戏页托盘、结算页落星、首页盒子与放大浮层（桌面 + 窄屏）。
