# 结算页峰终三段式 设计

日期：2026-06-11
状态：方向已确认，细节待评审；与 p12（角色）可合并实施

## 背景

结算页（`resultView`）现有落星进盒动画与累计星数展示。结尾缺少两件事：对今天学了什么的具体确认，和对明天的期待铺垫。

推测依据：峰终定律——体验记忆主要由峰值时刻与结尾决定；让孩子带着「我学会了 X」的具体成就感和「明天有 Y」的期待离开，比单纯的星星计数留存更好。通行结论（峰终定律本身有实验文献支撑，对幼儿场景的外推属推测），未逐条核验。

## 目标

结算页固定三段式编排：落星 → 点名今日所学 → 预告明天。全程约 8–12 秒，每段可点击跳过。

## 设计

### 第一段：落星（现状保留）

- 既有 `roundStars` 落盒动画与累计数不动，作为开场峰值。

### 第二段：点名今日所学

- 数据：当日 plan 的 `newItems`（无新内容日取本轮 `state.learnedItems` 去重后的前 3 项）。
- 表现：拼音大卡逐张翻出（最多 3 张），每张播一遍该条目读音 + 例词音（复用 `getTargetAudio` 路径），角色（p12）在旁做 `cheer` 表情。
- 无文字约束：卡上只有拼音 label 与插画，「你学会了」之意由角色庆祝动画与语音承担；若录制了角色点名音频（p12 的语音批次可加「你学会了」引导短句 + 条目音拼接），则播放，缺则只播条目音。

### 第三段：预告明天

- 数据：`getDailyPlan(明日 dayIndex)` 的首个 `newItems`；综合复习日/花园日则显示花园或复习主题图标。
- 表现：明日条目卡以剪影 + 问号呈现（不完全展示，留悬念），火车头朝它鸣笛一声；不播该条目读音（保留新鲜感）。
- 点击任意处回首页。

### 编排与降级

- 三段顺序由一个简单的 `state.resultPhase` 推进（定时器 + 点击跳过）；`prefers-reduced-motion` 时三段静态同屏展示、音频照常。
- 再练一轮（非当日首轮）只走第一段，避免重复仪式拖沓。

## 不做的事（YAGNI）

- 不做成绩统计展示给孩子（正确率是家长区的事）。
- 不做分享卡片。
- 第二段不超过 3 张卡（注意力预算内收尾）。

## 影响的文件

- `app.js`：`resultView`（三段结构与 phase 状态）、`handleClick`（跳过分发）、`nextQuestion`（首轮标记）。
- `styles.css`：翻卡、剪影问号、鸣笛动画、reduced-motion 静态布局。
- `assets/audio/`：可选的点名引导短句（随 p12 批次）。
- `scripts/verify-child-shell.js`：结算页断言更新（无可见中文长句保持）。

## 验证方式

- headless：三段 phase 依次推进且可跳过；再练一轮只出第一段；明日为复习日时第三段渲染复习图标。
- 人工：完整三段时长实测（目标 8–12 秒）；孩子是否理解剪影预告（实测观察是否引发「明天还要玩」的表达）。
