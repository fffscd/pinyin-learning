# 星星点亮兑换（不清零） 设计

日期：2026-06-11
状态：方向已确认，细节待评审

## 背景

星星现为终身累计、只增不减（2026-06-03 星星设计稿确认的原则）。累计数本身缺少增量目标——星星到了三位数后，多一颗的边际激励趋零。

推测依据：幼儿对失去已有奖励高度敏感，扣减式商店会引发抗拒；里程碑式「攒到就点亮」保留存量自豪感同时提供增量动机。通行结论，未逐条核验文献。

## 目标

星星总数永不减少的前提下，引入「点亮」体系：星星每攒满一个里程碑，获得一次点亮机会，用于点亮装饰物（火车车厢颜色、花园小动物等）。

## 设计

### 数据

- `progress.unlocks`：已点亮装饰 id 数组；`normalizeProgress` 补 `[]`。
- 点亮机会 = `floor(progress.stars / 20) - progress.unlocks.length`（每 20 颗星一次机会；参数为常量 `STARS_PER_UNLOCK`，实施时可按实际获星速率调整——当前每轮 5–8 星，20 星约 3 轮，频率适中）。
- 装饰目录常量 `UNLOCK_CATALOG`：每项 `{ id, kind, art }`，kind 分 `train-color`（车身配色，应用到 `trainArt`/p12 角色）与 `garden-pet`（花园常驻小动物，渲染进 `gardenView`）。首期 8–10 项。
- 应用状态：`progress.activeTrainColor` 记当前选用的车色（小动物全部常驻展示，无需选择）。

### 交互

- 入口：星星盒放大浮层（`starBoxOverlay`）内增加装饰架：已点亮项彩色可点（车色类点击即换色）、未点亮项剪影 + 一个小星标表示需攒星。
- 有可用点亮机会时：星星盒外圈发光提示；打开浮层后点任意剪影即点亮（点亮动画：星光注入、剪影上色），星星总数不变。
- 点亮顺序由孩子自由选，无稀有度、无随机。

### 原则边界

- 星星数字永不下降（界面任何处不出现减法）。
- 装饰均为纯视觉，不影响玩法与数据。
- 与 p09 花园植物的区别：植物是学习履历（自动生长、不可买），装饰是奖励消费品（点亮获得），两套并存不混淆。

## 不做的事（YAGNI）

- 不做随机抽取（盲盒机制对幼儿是赌博启蒙，明确排除）。
- 不做限时装饰、不做装饰下架。
- 不做装饰交易/赠送。

## 影响的文件

- `app.js`：`normalizeProgress`、`UNLOCK_CATALOG`、`starBoxOverlay`（装饰架）、`starBox`（发光提示）、`trainArt`（配色参数）、`gardenView`（小动物渲染）、`handleClick`。
- `styles.css`：装饰架、剪影态、点亮动画、星星盒发光、reduced-motion 兜底。
- `scripts/verify-child-shell.js`：浮层断言更新。

## 验证方式

- headless：机会数计算正确（stars=0/19/20/40 边界）；点亮后 `unlocks` 追加且 stars 不变；旧存档补默认。
- 人工：点亮动画、换车色即时生效（首页大卡与游戏页角色同步）、花园小动物常驻；窄屏浮层布局。
