# 首页单一主路径 设计

日期：2026-06-11
状态：方向已确认，细节待评审

## 背景

首页（`homeView`）目前并列 4 张玩法卡（`HOME_MODE_CARDS`：今天的学习 / 声调滑梯 / 拼音配图 / 打地鼠），外加花园入口、星星盒、家长角。孩子可以天天只点打地鼠，绕过课表设计的学习顺序。

推测依据：4–5 岁儿童面对并列选择易受位置和颜色支配而随机点击，或固守单一高刺激选项；减少并列入口可让课表编排真正被执行。通行结论，未逐条核验文献。

## 目标

首页改为单一主路径：「今日课程」一个大按钮占据视觉主体（首屏约 60%），声调滑梯与拼音配图收进花园作为普通游戏；课程内部由课表编排自动混入声调、配图等题型（现状 `buildCourse` 已支持）。

## 设计

### 结构调整

- `HOME_MODE_CARDS` 缩减为仅 `lesson` 一项，渲染为大卡：小火车主视觉（复用 `trainArt()`）+ 当日进度暗示（今日已完成则车厢亮灯）。
- `GARDEN_GAME_MODES` 增加两项：`tones`（声调滑梯，icon "tone"）与 `pictures`（拼音配图，icon "image"，与现有 `pinyin-pictures` 区分清楚：`pictures` 是听音选图，`pinyin-pictures` 是看拼音选图；若实测重合度高则合并为一项，实施时定夺）。
- 首页保留：大课程卡、花园入口（`gardenEntry`，尺寸略放大）、星星盒（`starBox`）、家长角（`parentCorner`）。

### 布局

- 桌面与窄屏均为纵向单列：课程大卡（约 60% 首屏高度）→ 花园入口横条 → 底部角落星星盒与家长角。
- 课程已完成的当天：大卡变为「到站」状态（烟囱冒彩烟），点击仍可再练一轮（`startRound("lesson")` 重新出题，现状已支持重复轮次）。
- 自由游戏日（p02）：大卡替换为花园主视觉，点击直达 `gardenView`。

### 行为守护

- 课程大卡是首页唯一的「开始学习」动作，点击直达 `startRound("lesson")`，不经过二级选择。
- 壳层无文字约束保持：大卡用图像表达，不新增可见中文长句；`verify-child-shell.js` 断言同步更新。

## 风险与对策

- 风险：孩子原本喜欢从首页直接打地鼠，入口下沉一级可能引起短期抗议。对策：花园入口放大并保留显著位置；打地鼠在花园内位置靠前。
- 风险：`pictures` 与 `pinyin-pictures` 玩法相近，花园出现两个相似入口。对策见上，实施时实测后定合并与否。

## 不做的事（YAGNI）

- 不做「今日任务清单」式的多步骤地图（一年后内容多了再议）。
- 不做时间锁（几点后才能玩花园之类）。
- 不删除任何玩法，只调整入口层级。

## 影响的文件

- `app.js`：`HOME_MODE_CARDS`、`GARDEN_GAME_MODES`、`homeView`、`homeModeCards`、`startRound`（tones/pictures 从花园模式进入的分支核对）。
- `styles.css`：课程大卡、首页单列布局、花园入口放大、窄屏适配。
- `scripts/verify-child-shell.js`：首页结构断言全面更新。
- `README.md`：首页描述更新。

## 验证方式

- headless：首页只渲染一张玩法大卡；花园渲染 8 个游戏入口；课程完成态切换正确；无可见中文长句断言通过。
- 人工：桌面与窄屏首页观感；课程完成前后大卡状态；从花园进声调滑梯与配图全流程。
