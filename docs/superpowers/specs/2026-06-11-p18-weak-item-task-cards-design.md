# 易忘项亲子任务卡 设计

日期：2026-06-11
状态：方向已确认，细节待评审；建议先实施 p15、p17

## 背景

「易忘」目前只是记录页的统计标签（`masteryBreakdown`、`updateDailyWeakItems`）。间隔重复全部发生在屏幕内、同一套题型语境中。

推测依据：跨情境提取（在生活场景里回忆屏幕上学的内容）比同界面重复测验的巩固效果更好（学习科学中提取练习与迁移的通行结论，未逐条核验文献）。

## 目标

家长区为每个易忘项生成一张可直接念出口令的线下任务卡，家长完成后一键回填，计入复习引擎。

## 设计

### 任务生成（新增 `buildOfflineTasks()`）

- 数据源：`masteryBreakdown` 易忘集合 + `getDueReviewIds` 当日到期项的交集优先，每天最多 2 张卡（家长负担上限）。
- 模板常量 `OFFLINE_TASK_TEMPLATES`，按条目类型分组，占位符填 `item.label` 与 `item.word`：
  - 韵母类：「散步时找 3 个圆圆的东西，每找到一个就一起说：yuán——」（模板内容按条目具体化）。
  - 声母类：「请孩子当小老师，教你读 {label}，你故意读错一次让他纠正。」（角色互换是高动机形式，推测依据：幼儿对「教大人」的胜任感需求，未核验文献。）
  - 音节/整体认读类：「吃饭时玩拼音接龙：你说 {label}，孩子找一个带这个音的词。」
- 同一条目当天模板确定性选取（`seededRandom`，种子=日期+条目 id），避免刷新换题。

### 回填

- 每张卡两个按钮：「完成了，孩子会」→ 按客观答对走 `updateLetterSchedule(id, true)`（升盒）；「完成了，还不熟」→ 按答错处理（享受 p03 当日保护）。
- 不设「跳过」按钮，未点即未完成，明日按当日数据重新生成。
- 回填记入当日 `dailyStat`，周报（p17）汇总「本周线下任务完成 N 个」。

### 呈现

- 位置：家长区周报下方「今天可以做」区块，`parent-theme` 样式。
- 卡片结构：条目大字 label + 任务句 + 两个回填按钮。无易忘项时区块隐藏。

### 数据诚实性边界

- 线下回填由家长主观判断，可靠性介于孩子自评与屏内客观答题之间。首版按客观答题同权处理（家长当面验证过），若后续发现家长普遍宽松，再降权——先简单，留观察点。

## 不做的事（YAGNI）

- 不做任务历史归档与连续打卡（避免给家长制造打卡焦虑）。
- 不做拍照/语音存证。
- 不做提醒推送。

## 影响的文件

- `app.js`：`buildOfflineTasks`、`OFFLINE_TASK_TEMPLATES`、`recordsView` 区块、回填分发与 `updateLetterSchedule` 接入。
- `styles.css`：任务卡样式。
- `scripts/verify-goal.js`：生成逻辑断言（易忘交集、每日上限 2、确定性选取、回填写盒）。

## 验证方式

- headless:任务数 ≤2 且来自易忘/到期集合；两种回填对盒子的影响正确；无易忘项时返回空。
- 人工：模板句念出来是否自然可执行（请真实家长试 3 张卡）；回填后记录页与周报数字联动。
