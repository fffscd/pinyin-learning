# 打地鼠节奏分级 设计

日期：2026-06-11
状态：方向已确认，细节待评审

## 背景

现状核实：打地鼠（`mole-choice` 题型）当前没有时限。地鼠以 `--rise-delay` 错峰冒出后常驻，答题机制与普通选择题相同（点选 → `answer()`），没有消失倒计时。规划原文假设的「限时压力」目前不存在，本设计相应调整为：默认档保持现状，新增可选的节奏档作为进阶挑战，并为节奏档设安全下限。

推测依据：幼儿抑制控制弱，限时压力下会放弃辨音改为乱拍；但适度的节奏感能提升投入度。通行结论，未逐条核验文献。

## 目标

打地鼠增加两档节奏：「慢慢玩」（现状，无时限，默认）与「小挑战」（地鼠停留一段时间后缩回，缩回后重新冒出，不判错）。档位按近期正确率自动推荐，孩子可随时退回慢档。

## 设计

### 档位规则

- `state.molePace`：`"calm" | "challenge"`，非持久化，进入打地鼠时计算默认值。
- 自动推荐：取最近 3 轮打地鼠正确率（从 `dailyStats` 或会话内统计），≥80% 时默认 `challenge`，否则 `calm`。
- 挑战档参数：地鼠停留 4 秒后缩回，停 1 秒再冒出；停留时间下限锁死 3 秒，不随正确率继续加速。缩回不算答错、不触发 `recordQuestionResult`，纯视觉节奏。
- 挑战档内连错 2 题立即自动退回 `calm`（本轮内生效），无任何提示音或惩罚表现。

### 交互与视觉

- 进入打地鼠的首屏加一个图标化档位开关（乌龟图标=慢慢玩，兔子图标=小挑战），不出现中文长句。
- 缩回/冒出动画用 CSS 实现，`prefers-reduced-motion` 时挑战档退化为透明度切换。
- 点中缩回中的地鼠不判定（`pointer-events` 关闭于缩回动画期间），避免误触判错。

### 实现位置

- `renderMoleChoice`/`moleClass` 增加缩回状态类；冒出/缩回循环由 CSS animation 驱动（`animation-iteration-count: infinite` + 关键帧停留），避免 JS 定时器管理。
- 正确率统计：会话内维护 `state.moleRecent = []`（每题对错），自动降档逻辑在 `answer()` 的 mole 分支后处理。

## 不做的事（YAGNI）

- 不做三档以上、不做地鼠加速曲线。
- 不做超时判错（缩回永远只是视觉事件）。
- 不持久化档位偏好（每次按近期表现重新推荐）。

## 影响的文件

- `app.js`：`startRound`（moles 分支初始化档位）、`answer`（降档判定）、`renderMoleChoice`、`questionStage`（档位开关）、`handleClick`（开关分发）。
- `styles.css`：缩回/冒出关键帧、档位开关、reduced-motion 兜底。

## 验证方式

- headless：正确率高低两种构造下默认档位正确；连错 2 题后档位变 `calm`；缩回事件不产生答题记录。
- 人工：两档在桌面与触屏的节奏观感；3 秒下限是否仍来得及辨音（请目标年龄孩子实测一轮）；reduced-motion 表现。
