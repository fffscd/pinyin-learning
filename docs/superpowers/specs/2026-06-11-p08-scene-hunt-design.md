# 拼音寻宝场景玩法 设计

日期：2026-06-11
状态：方向已确认，细节待评审；依赖场景插画（p19 产能）

## 背景

现有配对类玩法（拼音配图、听词找拼音）都是孤立卡片对卡片。拼音与真实生活语境的联结只靠 `word` 字段的例词音频。

推测依据：把目标音放回有意义的场景中检索，比无语境卡片配对形成更强的「声音—符号—生活图像」三联结（情境化学习的通行结论，未逐条核验文献）。

## 目标

新增花园玩法「寻宝」：一幅大场景图（如厨房、公园）里藏 4–5 个可发声物品，播放拼音或例词读音，孩子在图中找到并点击对应物品。

## 设计

### 数据

- 新增 `SCENE_HUNT_SCENES` 常量：

```js
{
  id: "kitchen",
  label: "厨房",
  art: "scene-kitchen",            // 场景 SVG 标识
  spots: [
    { itemId: "guo", x: 32, y: 58, w: 14, h: 12 },  // 百分比定位的热区
    ...
  ],
}
```

- 热区用百分比坐标 + 宽高，场景图等比缩放时热区跟随，窄屏不需要单独适配数据。
- 每个 spot 关联 `PINYIN_ITEMS` 中 `pictureable` 的条目；首期一张场景，物品全部取自首月已有 SVG 插画的条目，不等新插画。

### 玩法流程

1. 进入后整图亮相，自动逐个聚光介绍一遍物品并播例词音（建立场景词汇，约 10 秒，可点击跳过）。
2. 出题：播目标读音（`AUDIO_PROMPTS` 沿用听音找拼音的提示音逻辑），孩子点场景中的物品。
3. 点对：物品弹跳 + 高亮 + 例词音，记星；点错：走统一重试流程（p10），重播目标音。
4. 一轮把场景内全部 4–5 个物品找完即结算；找完的物品留下发光标记，强化「都被我找到了」的完成感。
5. 结果照常走 `recordQuestionResult` 回写 Leitner。

### 入口与扩展

- `GARDEN_GAME_MODES` 增加 `mode: "hunt"`（图标：放大镜）。
- 场景按月扩充（公园、超市、动物园），每张覆盖当月新引入的 pictureable 条目；场景选择按已解锁条目自动过滤，无可用场景时入口隐藏。

### 触控

- 热区实际可点范围在视觉物品外扩 8%（幼儿点不准物品边缘），相邻热区不重叠；遵守 p13 的最小目标规格。

## 不做的事（YAGNI）

- 不做场景内自由探索彩蛋（首期克制，验证核心玩法）。
- 不做拖拽交互。
- 不为场景图配背景音乐。

## 影响的文件

- `app.js`：`SCENE_HUNT_SCENES`、`makeSceneHuntQuestion`、`buildSceneHuntQuestions`、`questionStage` hunt 分支、`GARDEN_GAME_MODES`、`handleClick` 热区分发。
- `styles.css`：场景容器、热区、聚光与发光标记动画。
- 场景 SVG：内联或独立 symbol，复用现有插画风格（p19 锁定的规范）。

## 验证方式

- headless：题序覆盖全部 spots；点错走重试；全部找完进结算。
- 人工：桌面与窄屏下热区与视觉物品对位（旋转、缩放窗口）；介绍环节可跳过；目标年龄孩子能否在 5 物品场景中独立完成。
