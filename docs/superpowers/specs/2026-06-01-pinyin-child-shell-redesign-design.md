# 拼音学习 · 面向独立玩的语音优先外壳重构设计

日期：2026-06-01
状态：已与产品负责人确认设计，待转实施计划

## 1. 背景与目标

`pinyin-learning` 是面向幼儿园中班儿童的拼音启蒙 Web 小游戏，已实现 30 天课程、5 个花园游戏、声调滑梯、自由练习和本地进度记录。本次重构基于一次以「5 岁、零识字」视角的复审，发现现有外壳存在阻断独立使用的问题。

确认的四个前提（已与产品负责人逐项确认）：

1. 使用场景：**孩子独立玩**（家长不在旁边）。这把「近乎零文字 + 全程语音指令 + 图标导航」从优化建议升级为硬要求。
2. 重做范围：**重构体验壳**。保留题库与课程逻辑，重做面向幼儿的语音优先外壳（首页、导航、反馈、素材体系）。
3. 素材方向：**改良手绘 SVG 插画**。用统一风格的内联 SVG 替换 emoji，无外部依赖、跨设备一致。
4. 音频策略：**沿用现有 TTS，结构预留真人录音**。本次不重录音频，保持 manifest 驱动的可替换结构。
5. 导航模型：**单主线 + 奖励抽屉**。

本设计的成功标准：一个不识字的 5 岁儿童，在零阅读前提下，能独立从首页进入并完成一轮课程。

## 2. 复审发现的问题（重构依据）

可验证问题（基于代码与文档）：

- 配图用 emoji 占位，未达需求文档第 7 节「清晰、单一主体」标准；部分 emoji 与联想词关联很弱（`de`→✅、`ne`→💬）。
- 联想词部分不适龄：`de=得到`、`ne=呢喃`、`te=特别`、`fo=佛塔` 对 5 岁儿童无生活意义。
- 文字指令偏多：游戏标题是整句中文（`getGameTitle` 的「听声调，选轨迹」等），花园入口是文字名。
- 首页入口过多：`homeView()` 有两组重复入口（hero 区 与「入口」区）加「游戏」区 5 张卡，对不识字孩子是 10+ 个入口。
- 声母单独听辨偏难：纯听 `b`/`p` 并选字母涉及送气细微差别，是 5 岁高难任务。

需核实/推测的问题：

- 音频质量：仓库已有 TTS 生成的 `.webm`，声母单读音用 TTS 通常不准。（推测，依据：README 说明与 `.tmp/MeloTTS` 产物）
- 触屏按钮尺寸与误触保护：待实施阶段结合 `styles.css` 核实。

## 3. 设计

### 3.1 导航与首页：单主线 + 奖励抽屉

**删除**：`homeView()` 中两组重复入口（hero 的 开始/āá/看图、「入口」区的 课程/āá/看图/练习/记录）和「游戏」区。

**新首页只保留三件元素**：

- 顶部固定大喇叭按钮：再听一次语音指令。
- 火车插画（沿用或改良 `trainArt()`）。
- 占屏底部的巨大「开始」按钮：直接进入今日课程。

进入首页自动播放语音指令（「今天我们学 X」）。

**奖励抽屉**：完成今日课程后，首页与结算页出现「游戏花园」单一大图标入口；进入后是现有 5 个花园游戏，全图标化 + 进入时语音报名。未完成今日课程时，花园入口灰化并显示小锁图标，点击不报错、不出现挫败式提示。

**降级入口**：

- 声调、看图并入课程流程，不再做独立首页入口。
- 自由练习并入花园。
- 学习记录与静音设置移到「家长角」：放在屏幕角落的小区域，通过长按进入，降低孩子误入概率。

涉及代码：重写 `homeView()`；新增花园抽屉视图或复用现有花园渲染；调整 `setView` 与首页事件绑定；新增家长角入口与长按交互。

### 3.2 语音指令系统（voice-first）

- 进入任何界面自动播放该界面对应语音指令，复用并扩展现有 `playCurrentPrompt`。
- 顶部固定大喇叭 =「再听一次」，任何时候可点，复用现有 `volume`/`repeat` 图标。
- 点击任意拼音卡或选项时先朗读其读音。
- 屏幕上的中文整句指令（`getGameTitle` 返回值等）从视觉层移除，降级为 `sr-only`（保留无障碍语义），改由语音 + 图标承担任务表达。
- 静音设置归入家长角，默认开启声音。

### 3.3 零文字反馈

- 答对：火车前进一节或花朵绽放动画 + 音效 + 语音鼓励，不依赖文字。
- 答错：温和音效 + 重播读音 + 正确项轻微高亮，不出现文字提示。
- 现有 `state.feedback` 文本降为 `sr-only` 或移除其视觉呈现。
- 保留需求文档 4.4 的阶段奖励：每 3–5 题增加一节车厢；完成当日课程给毕业动画。

### 3.4 SVG 插画素材体系 + 联想词修订

新增 `illustration(itemId)` 函数：返回统一画风（粗描边、明快配色、单一主体）的内联 SVG，替换 `pictureArt()` 中的 emoji 渲染。

**联想词与配图策略**（教学法事实，非个人偏好）：`de/te/ne/le/fo` 这类音节难以找到贴近幼儿生活的单图词。因此配图只用于单韵母和有好词的音节；纯声韵组合中的难配图音节只做听辨，不强配图。

配图保留并绘制 SVG 的条目：

- 单韵母：`a` 阿姨、`o` 公鸡、`e` 白鹅、`i` 衣服、`u` 乌云、`ü` 小鱼。
- 声母：`b` 爸爸、`p` 苹果、`m` 妈妈、`f` 风车、`d` 小鼓、`t` 小伞、`n` 奶奶、`l` 气球。
- 音节：`ba` 爸爸、`pa` 爬坡、`ma` 妈妈、`fa` 发芽、`bo` 菠萝、`po` 婆婆、`mo` 蘑菇、`da` 大鼓、`ta` 宝塔、`na` 拿球、`la` 喇叭。

仅做听辨、不配图的条目：`de`、`te`、`ne`、`le`、`fo`。

实现方式：在 `PINYIN_ITEMS` 增加一个布尔标记（如 `pictureable`），在生成配图题的 `makePictureQuestion` / `makePinyinPictureQuestion` 及其题源里过滤掉 `pictureable === false` 的条目。这是对题目生成逻辑的受控小改动，是第 3.6 节「保留不动」的明确例外。

`l` 的联想词由「快乐」改为「气球」（具象、可画、与现有 🎈 一致）。`le` 保留为听辨项。

### 3.5 教学法微调（本次纳入）

- 声母易混对（`b/p`、`d/t`、`n/l`）首次出现时降到 2 选项，并用带元音的延长示范（如 `bo`/`po`）辅助，避免纯声母直接 3 选 1 造成挫败。涉及 `makeChoices` 在声母易混对场景下的选项数与示范音处理。
- 声调具象化：滑梯轨迹配一个具象物体动画（如小车沿轨迹上/下坡），点击播放声音并触发动画。涉及 `toneArt` 与声调题渲染。

### 3.6 明确保留不动

以下保持不变：题库 `PINYIN_ITEMS` / `TONE_ITEMS` 的核心结构（仅 3.4 增加 `pictureable` 标记、修订个别词）、`COURSE_PLAN_30_DAYS`、出题函数主体、`progress` 存储、audio manifest 与 TTS 结构。

本次只改「壳」：`homeView`、导航与事件绑定、反馈渲染、`pictureArt`、文案与语音指令、SVG 素材，以及 3.4/3.5 列出的受控小改动。

## 4. 受影响的代码单元

- `homeView()`：重写为单主线首页。
- 新增花园抽屉视图 / 家长角入口。
- `setView` 与事件绑定：适配新导航与长按。
- `getGameTitle` 等文字指令：降级为 `sr-only` + 语音。
- `playCurrentPrompt` / 语音调度：扩展到首页与导航进入时。
- `pictureArt()` → 新增 `illustration(itemId)` SVG 渲染。
- `PINYIN_ITEMS`：增加 `pictureable` 标记、修订 `l` 等个别词。
- `makePictureQuestion` / `makePinyinPictureQuestion`：按 `pictureable` 过滤。
- `makeChoices`：声母易混对降难。
- `toneArt` 与声调题：具象化动画。
- 反馈渲染与 `state.feedback`：零文字化。
- `styles.css`：首页、喇叭、按钮尺寸、家长角、动画、SVG 样式。

## 5. 错误与边界处理

- 音频缺失或加载失败：沿用现有 manifest 缺失处理，界面不阻断，仍可点击作答（已有 `missingAudio` 机制）。
- 花园未解锁：点击锁定入口仅播放轻提示语音，不报错。
- 家长角长按：需防止孩子普通点击误入；长按阈值与触屏兼容在实施时确定。
- 进度数据：沿用现有本地存储与 server/localStorage 双路径，无结构破坏性变更。

## 6. 测试与验收

- 核心验收：不识字的 5 岁儿童在零阅读前提下，能独立从首页进入并完成一轮课程。
- 每个界面进入都有语音指令；屏幕上无构成阻断的中文整句指令。
- 配图条目均为 SVG，跨主流浏览器渲染一致；`pictureable === false` 的条目不出现在配图题中。
- 声母易混对首现为 2 选项；声调题有具象动画。
- 现有 `scripts/verify-goal.js`、`scripts/verify-audio-assets.js` 仍通过。
- 手动：平板触屏下按钮尺寸与点击准确性、家长角不易误入。

## 7. 不在本次范围

- 课堂大屏模式、离线能力、家长/老师远程记录视图。
- 题库与 30 天课程内容的扩充。
